# 摔倒检测优化说明

## 概述
本次优化将原来的"单帧规则判断"升级为"姿态评分 + 连续帧状态机"，大幅降低误报率和重复报警。

## 核心改进

### 1. 单帧姿态评分 (而非直接报警)
- **之前**: 只要某一帧满足类似"鼻子低于肩膀"就直接报警
- **现在**: 每帧计算0.0 ~ 1.0的摔倒分数，综合多个指标

### 2. 连续帧确认机制
- 最近15帧中至少8帧是疑似摔倒(分数≥0.65)
- 持续时间≥1秒
- 避免了一两帧姿态误识别导致的误报

### 3. 按人维护状态
使用状态机管理每个人的状态:
- **normal**: 正常站立
- **suspect**: 疑似摔倒(高分数持续中)
- **confirmed**: 确认摔倒(报警触发)
- **cooldown**: 冷却期(不重复报警)
- **recovered**: 已恢复

### 4. 报警策略优化
- 同一个人在30秒内只报警一次
- 只有在状态从suspect变为confirmed时才报警
- 报警冷却时间从5秒延长到30秒

### 5. 关键点质量过滤
- 左右肩至少一个可见(置信度>0.5)
- 左右髋至少一个可见
- 可见关键点数量≥6
- 低质量关键点直接返回0分，不参与判断

## 分数计算因子

| 因子 | 最大加分 | 说明 |
|-----|--------|-----|
| aspect_ratio | +0.25 | 人体框高宽比变低(横躺) |
| compressed_trunk | +0.25 | 关键点整体高度/宽度变低 |
| horizontal_body | +0.20 | 肩膀-髋部方向接近水平 |
| head_low | +0.15 | 头部明显低于肩膀/髋部 |
| fast_down | +0.20 | 身体中心快速下移 |
| knee_spread | +0.10 | 膝盖间距异常大 |
| ankle_spread | +0.10 | 脚踝间距异常大 |
| quality_penalty | -0.30 | 关键点质量不足的惩罚 |

## 参数配置

| 参数 | 值 | 说明 |
|-----|----|-----|
| pose_conf_threshold | 0.5 | 关键点置信度阈值 |
| fall_score_threshold | 0.65 | 摔倒分数阈值 |
| confirm_window_frames | 15 | 确认窗口帧数 |
| confirm_min_fall_frames | 8 | 最小疑似帧数 |
| confirm_min_duration | 1.0秒 | 最小持续时间 |
| recovery_duration | 2.0秒 | 恢复判定时间 |
| cooldown_seconds | 30 | 冷却期时间 |

## 新文件

### `api/fall_detector.py`
- `FallState`: 状态枚举
- `PersonState`: 个人状态管理
- `FallStateTracker`: 摔倒追踪器(核心类)

## 修改的文件

1. `api/main.py`: 集成新的摔倒检测器
2. `api/event_deduplicator.py`: 更新摔倒去重时间

## API接口

新增 `/api/fall/status` 端点，可获取:
- 当前追踪的人数
- 活跃的摔倒事件
- 每个人的详细状态
- 阈值配置信息

## 视觉反馈

- **绿色框**: 分数 < 0.3 (正常)
- **橙色框**: 0.3 ≤ 分数 < 0.5 (中等)
- **红色框**: 分数 ≥ 0.5 (高风险)
- 框上显示具体分数值

## 改进效果

1. **误报率降低**: 避免弯腰、下蹲、遮挡被误判
2. **重复报警消除**: 30秒内同一人只报警一次
3. **更准确的判断**: 综合多个指标和时间序列
4. **可调试性增强**: 提供分数和状态信息
